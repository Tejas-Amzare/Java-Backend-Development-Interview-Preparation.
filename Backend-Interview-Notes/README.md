# 🚀 Java Backend Development — Interview Preparation Notes

Simple, clean, beginner-friendly study notes for **Java Backend Developer** interviews.
Every topic is written in **easy English**, with examples, diagrams, interview questions,
common mistakes, best practices, and a **quick revision summary**.

> 🎯 Goal: Read → Understand → Revise → Crack the interview.

---

## 📁 Folder Structure

```
Backend-Interview-Notes/
│
├── Java/               → Core Java, OOP, Collections, Threads, JVM, Streams
├── Spring/             → Spring Core, IoC, DI, Beans, AOP
├── SpringBoot/         → Spring Boot, REST API, Actuator, Caching
├── Hibernate/          → JPA, ORM, Relationships, N+1, Performance
├── SQL/                → CRUD, Joins, Normalization, Transactions, Windows
├── DSA/                → Arrays, Trees, Graphs, DP, Complexity + LeetCode
├── Security/           → Auth, JWT, OAuth2, Filters, CORS/CSRF
├── Microservices/      → Gateway, Discovery, Kafka, Circuit Breaker
├── Docker/             → Images, Containers, Compose, Volumes
├── Kubernetes/         → Pods, Deployments, Services, Ingress
├── AWS/                → EC2, S3, RDS, IAM, Lambda, ECS
├── Git/                → Branching, Merge, Rebase, Git Flow
├── SystemDesign/       → HLD, LLD, Scalability, Caching, CAP
├── Projects/           → Full backend architectures (8 projects)
├── HR/                 → HR round questions + answers
├── CompanyWise/        → Amazon, Google, TCS, Infosys, etc.
├── CheatSheets/        → 1-page quick revision sheets
├── InterviewQuestions/ → 500+ Q&A across all topics
└── README.md           → You are here
```

## 🗺️ How to Use These Notes

1. **First time?** Read a topic fully (Concept → Example → Diagram).
2. **Revising?** Jump straight to the **Quick Revision** at the end of each file.
3. **Day before interview?** Read only the `CheatSheets/` folder.
4. **Targeting a company?** Read `CompanyWise/` for that company.
