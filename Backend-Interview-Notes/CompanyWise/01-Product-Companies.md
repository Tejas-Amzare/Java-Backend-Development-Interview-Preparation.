# Product Companies — Interview Questions

> Focus: **Strong DSA + System Design + Java fundamentals.** Multiple coding rounds.

---

## 🟠 Amazon
**Focus:** DSA (medium-hard), System Design, **Leadership Principles** (very important!).
- Two Sum, LRU Cache, Number of Islands, Merge K Sorted Lists.
- Design a URL shortener / Amazon cart / order system.
- Behavioral (STAR): "Tell me about a time you disagreed with your manager" (Ownership, Dive Deep,
  Customer Obsession).
- Java: HashMap internals, multithreading, GC.
- SQL: 2nd highest salary, joins.
> 💡 Prepare **STAR stories** mapped to Amazon's 16 Leadership Principles.

## 🔵 Google
**Focus:** Algorithms + data structures, clean code, edge cases.
- Trees, graphs (BFS/DFS), DP (edit distance, coin change), backtracking.
- Design: rate limiter, autocomplete (Trie), Google Docs.
- Big-O analysis is a must; discuss trade-offs.
> 💡 Communicate your thought process out loud; handle edge cases.

## 🟩 Microsoft
**Focus:** DSA (linked lists, trees, strings), OOP design, problem-solving.
- Reverse linked list, detect cycle, LCA, serialize a tree.
- OOP: design a parking lot / elevator (LLD).
- Java: OOP concepts, collections, exceptions.

## 🔴 Oracle
**Focus:** Core Java, DBMS/SQL (heavy), DSA.
- SQL: joins, normalization, indexing, query optimization.
- Java: JVM, multithreading, collections.
- DSA: arrays, strings, sorting.

## 🔵 Walmart
**Focus:** DSA + System Design + Spring Boot (backend roles).
- Medium LeetCode (arrays, hashing, trees).
- Design an e-commerce/inventory system.
- Microservices, REST API design, Kafka.

## 🟠 Flipkart
**Focus:** DSA (hard), System Design, low-level design.
- Graphs, DP, sliding window.
- Design: Flipkart cart, notification system, order management.
- Concurrency, caching, scalability.

## 🟢 Swiggy / Zomato
**Focus:** DSA + System Design (real-time, location-based).
- Design: food delivery system, real-time order tracking, restaurant search.
- Handling high traffic, caching, geo-queries.
- Java concurrency, Spring Boot, Kafka.

## 🔵 Razorpay / PhonePe (Fintech)
**Focus:** DSA + System Design + reliability + concurrency.
- Design: payment gateway, wallet, idempotent transactions.
- Concurrency, ACID, distributed transactions, idempotency keys.
- Java internals, low-latency thinking.
> 💡 Emphasize **correctness, consistency, and idempotency** — money must never be wrong.

## 🎯 Tips
- Think out loud; state complexity.
- Clarify requirements before coding.
- Handle edge cases and test your code.
- For fintech: stress consistency & idempotency.
